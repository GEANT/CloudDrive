Linux
On Linux, most UI clients simple don't work with webdav over ssl very well. Luckily, there is a package that allows you to mount cloud backed storage as a normal disk. We'll leave permissions and groups outside the scope of this quick walk through - basically it is just any other Linux filesystem, so you should do what you always do.
Use your system's package manager to install a package named davfs2. Once you have done that, you MUST edit the configuration file for davfs2 (on Ubuntu it's located in /etc/davfs2/davfs2.conf).


NOTE: RHEL5 users wil want to use the RPM http://pkgs.org/centos-5-rhel-5/rpmforge-x86_64/davfs2-1.4.6-1.el5.rf.x86_64.rpm.html


Uncomment the line:

if_match_bug 0

and change it to:

if_match_bug 1

If you do NOT do this, davfs2 will not work for you with cloud backed storage. While we're at it, you might want to enable locking - a bit safer when multiple computers access a file. So uncomment the line:

use_locks 0 and change it to use_locks 1

Now create a mount point, say /mnt/clouddrive and type:

mount -t davfs https://rightfabric.net/ /mnt/clouddrive

Supply your username and your password here, accept the certificate (it's a “real” one, but strangely enough you sometimes get a warning).
And that's all! /mnt/clouddrive now maps to cloud backed storage.
